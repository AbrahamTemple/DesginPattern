# 桥接模式

## 简单桥接

> 电脑抽象里有CPU接口作为成员变量

``` bash  
├── 硬件
   ├── 电脑
   |    ├── 神州 ──────── +   
   |    └── 联想 ──── +   |   
   └── CPU            |   |   
        ├── AMD  ──── +   |   
        └── Intel ─────── +   
```

## 多路桥接

> 这里结构其实跟上面没区别，只是引用时灵活了一点

``` bash  
├── 存储
   ├── 存储目的地
   |    ├── 云端存储 ───────── +   
   |    └── 本地存储 ──── + ── +   
   └── 存储方式           |    |   
        ├── 数据库存储 ── + ── +   
        └── 文件存储 ───────── +   
```
